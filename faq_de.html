FAQ


Was ist Cloud Computing?

Cloud Computing bedeutet, dass Sie Ihren Datenbanken oder Speicher, Ihre Anwendungen und andere Computing-Dienste über einen Cloud Provider anmieten. Sie zahlen dabei nach Nutzung, also bsw. nur für Server, die auch laufen.


Was ist AWS?

Amazon Web Services ist mit mehr als 200 Services, seit Jahren die weltweit umfassendste und am häufigsten genutzte Cloud-Plattform.


Warum in die Cloud wechseln?

Sie können in der Cloud Änderungen und Erneuerungen von ihrem Geschäftsmodell wesentlich schneller und flexibler durchführen, Ressourcen dynamisch erhöhen oder verringern, außerdem ihre Investionskosten für Hardware vermeiden, stattdessen bezahlen sie nur, was sie wirklich nutzen.

Spare ich Geld mit dem Wechsel in die Cloud?

Falls sie ihre bestehende OnPremise Systemlanschaft einfach nur 1:1 in die Cloud kopieren, werden sie vermutlich keine Ersparnisse erzielen können. Häufig sind OnPremise System vertikal überdimensioniert und werden effektiv nur zu 20% genutzt. Wenn sie die Cloud jedoch korrekt für ihre Zwecke einsetzen, richtig skalieren, moderen Serverless Architekturen nutzen, können sie definitiv Kosten sparen, insbesondere in der Administration.

Kann ich selbst in die Cloud wechseln?

Falls sie für die Cloud bzw. AWS gut geschultes IT Personal haben, können sie natürlich selbst in die Cloud migrieren. Falls sie und ihre Mitarbeiter gerade erst ihren ersten AWS Account registriert haben und die ersten Gehversuche unternehmen, raten wir Ihnen, holen sie sich professionelle Hilfe. Zumindest für eine Weile, bis die wichtigsten Services migriert sind und sie sich Best Practices abschauen konnten.

AWS oder Azure? Welche Cloud sollte ich nutzen?

Machen sie eine Analyse ihrer bestehenden Serverlandschaft. Welche Server wollen sie in die Cloud migrieren? Desktop Systeme können sie hierbei ignonieren, die werden sie nicht migrieren. Falls sie größtenteils Windows Server nutzen, macht eine Migration zu Azure durchaus Sinn. Falls sie jedoch hauptsächlich Linux Server betreiben, behalten sie diese bitte und migrieren zu AWS.

AWS oder Azure, wer ist billiger?

Die Preise für virtuelle Maschinen und Compute Services sind für beide Provider relativ ähnlich. Falls sie eine identische Systemlanschaft auf AWS oder Azure betreiben, werden sich die Kostenunterschiede in Grenzen halten. Azure Kunden nutzen allerdings häufiger Windows Server anstatt Linux, Microsoft Datenbanken anstatt MySQL oder Postgres und generieren auf diese Weise höhere Hardware und insbesondere Software Lizenz Kosten

Wer hilft mir meine bestehende AWS Architektur zu verbessern?

SCC Brasil macht Architektur Reviews um bestehene AWS Systemlandschaften zu prüfen. Dabei erstellen wir einen Bericht mit allen Findings und somit Verbesserungs-möglichkeiten und -maßnahmen. Hauptfokus sind hier Kostenersparnis, Sicherheit, Ausfallsichereit & Nutzung von Best Practices.

Wie spare ich Kosten in der Cloud? Wie spare ich Kosten auf AWS?

Sie können mit einigen klugen Entscheidungen enorm Kosten auf AWS sparen. Generell: Nutzen den richtigen Instance Type & Size. Skalieren sie horizontal und nicht vertikal. Stoppen Sie Instanzen, die nicht benötigt werden, bsw. ausserhalb der Geschäftszeiten. Sparen sie mit Saving Plans und Spot oder Reserved Instances. Nutzen Sie Serverless und vermeiden Sie Transferkosten mit VPC Endpoints. Wir beraten Sie hierbei gerne.

Wie migriere ich in die Cloud? Wie migriere ich nach AWS?

Das hängt davon ab, was exakt sie in die Cloud Migrieren wollen. Speicher, Backup, Applikationen, Datenbanken usw. Für die Migration ihrer Geschäftslösungen gibt es verschiedene Strategien, auch bekannt unter die 7 R's. refactor, replatform, repurchase, rehost, relocate, retain, and retire. Sie müssen für sich die beste Strategie wählen. Sie finden hierzu mehr Informationen in unserem Blog.

Ist die Cloud auch sicher?

Richtig konfiguriert bietet AWS und die Cloud im Allgemeinen eine hervorragende Sicherheit und sogar noch mehr Möglichkeiten ihre Daten zu schützen als On Premise. Allerdings steckt eine Menge KnowHow in diesem Thema und sie müssen Best Practices folgen. Letztendlich liegt die Verantwortung bei Ihnen, die entsprechenden Sicherheits Maßnahmen richtig einzusetzen. Lesen sie mehr darüber in unserem Blog:
Security Best practices on AWS

Benötige ich einen AWS Account oder mehrere?

Auch dies hängt von Ihrem Geschäftsmodell ab, allerdings empfehlen wir die Nutzung mehrerer Accounts und die strikte Trennung und somit Isolierung von verschiedenen Stages, Umgebungen oder Kunden-Lösungen. Sie finden hierzu mehr Informationen in unserem Blog.
